# Identity Primitive Specification

## Normative Scope

**Artifact:** ECP-RP-v0.1.0-E0.2.1
**Parent:** ECP-RP-v0.1.0-E0.2 (Primitive Schema Foundation)
**Level:** L0 (Foundation Primitive)
**Dependencies:** None

---

## 1. Purpose

Define the canonical representation of immutable identifiers, canonical URIs,
provenance lineage, and orthogonal versioning for all governed objects within
the LEP portfolio. Identity is the foundational primitive upon which all other
schemas depend. Every constitutional property established here propagates to
every downstream schema.

---

## 2. Constitutional Properties

These five properties are frozen at the constitutional level. They cannot be
amended without a Portfolio Authority evolution request affecting all levels:

### 2.1 Dual Identity Model

Every governed object carries two identity fields:

- `id` — Stable logical identifier. Used for internal references within the
  portfolio. Does not require URI resolution. Consumers may reference objects
  by `id` without constructing full URIs.

- `uri` — Canonical globally unique identifier. Resolves to the authoritative
  representation of the governed object. Every `uri` is an absolute URI
  conforming to RFC 3986.

**Rule:** `id` provides logical identity. `uri` provides canonical global
identity. They are both required and both immutable after publication.

### 2.2 Orthogonal Versioning

Version is not embedded in identity. An object's identity (`id`, `uri`) does
not change when its version changes. Version is a separate, mutable property
on the identified object.

```
CORRECT:                  INCORRECT:
  id: artifact              id: artifact-v1
  version: 1.0.0
```

**Rule:** Identity and version are orthogonal. Versioning an object never
changes its identity. Identity changes require a new artifact, new URI, and
new lineage.

### 2.3 Immutability

Once published, identity is immutable. Every identity carries:

```json
"immutable": true
```

with `const: true` enforced by schema validation. Any document that sets
`immutable: false` or omits it fails schema validation.

**Constitutional effect:** If identity must change — create a new artifact
with a new URI. The original identity remains frozen. The old artifact's
`lineage.superseded_by` points to the new artifact's URI.

### 2.4 Lineage

Every governed object supports provenance tracking through an explicit
lineage model:

```yaml
lineage:
  parent: null              # URI of parent object, or null if root
  supersedes: []            # URIs of objects this replaces
  superseded_by: null       # URI of object that supersedes this, or null
```

Lineage enables governance history without polluting identity. A root object
(the first publication) has `parent: null` and `supersedes: []`. A successor
has `parent: <original-uri>` and `supersedes: [<original-uri>]`.

### 2.5 Canonical URI Template

All canonical URIs follow a single constitutional template:

```
https://specs.lysergic.dev/{domain}/{kind}/{name}
```

Where:

| Component | Description | Example |
|-----------|-------------|---------|
| `domain`  | ECP program domain | `primitives`, `ecp`, `evidence` |
| `kind`    | Artifact classification | `v1` (schemas), `E0.2.1` (artifacts) |
| `name`    | Object-specific identifier | `identity.schema.json`, `identity` |

This template is constitutional. Future repository reorganizations must not
change published URIs. Only Portfolio Authority may extend the template.

## 5. Pydantic v2 Mapping Guidance

When generating Pydantic v2 models from this schema:

```python
from pydantic import BaseModel, Field
from typing import Optional
from datetime import datetime

class Lineage(BaseModel):
    parent: Optional[str] = None
    supersedes: list[str] = Field(default_factory=list)
    superseded_by: Optional[str] = None

class Identity(BaseModel):
    id: str = Field(description="Stable logical identifier")
    uri: str = Field(description="Canonical globally unique identifier")
    version: str = Field(
        default="0.1.0",
        pattern=r"^[0-9]+\.[0-9]+\.[0-9]+$",
        description="Semantic version (orthogonal to identity)"
    )
    immutable: bool = Field(
        default=True,
        const=True,
        description="Constitutional invariant — always true"
    )
    lineage: Optional[Lineage] = Field(default=None)
```

The `immutable` field uses `const=True` which Pydantic v2 enforces as a
frozen value. Any attempt to set `immutable=False` raises a validation error.
